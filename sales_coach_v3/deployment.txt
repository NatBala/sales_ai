Step 1 — Create deploy zip (MUST use Python, NOT PowerShell Compress-Archive):
  python -c "
  import zipfile, os
  with zipfile.ZipFile('deploy.zip', 'w', zipfile.ZIP_DEFLATED) as zf:
      for dirpath, dirnames, filenames in os.walk('app'):
          dirnames[:] = [d for d in dirnames if d != '__pycache__']
          for filename in filenames:
              if filename.endswith('.pyc'):
                  continue
              filepath = os.path.join(dirpath, filename)
              arcname = filepath.replace(os.sep, '/')
              zf.write(filepath, arcname)
      zf.write('requirements.txt', 'requirements.txt')
  print('deploy.zip created')
  "

  !!! Do NOT use PowerShell Compress-Archive !!!
  It stores paths with backslashes (app\main.py) which on Linux are treated as
  literal filenames, not directory separators, breaking all Python imports.

Step 2 — Deploy:
  az webapp deploy -g cg_irvine -n coachme-vg --src-path deploy.zip --type zip

  Expected output: "RuntimeSuccessful" with numberOfInstancesSuccessful: 1

  Note: The CLI may report "Site failed to start" due to a 10-minute warm-up
  timeout, but the app usually starts successfully. Confirm with health check.

Step 3 — Verify:
  curl https://coachme-vg.azurewebsites.net/api/health
  Expected: {"status":"ok","openai_configured":true}
