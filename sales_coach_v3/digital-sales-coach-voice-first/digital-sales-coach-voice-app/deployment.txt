DEPLOYMENT INSTRUCTIONS — coachme-vg
=====================================

App URL: https://coachme-vg.azurewebsites.net
Resource Group: cg_irvine
App Service Plan: cg-voice-agent-plan (P1v2, West US 2)
Runtime: Python 3.10 (Linux)
Startup Command: gunicorn -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000 app.main:app


LOCAL SETUP
-----------
1. Navigate to the app directory:
   cd digital-sales-coach-voice-first/digital-sales-coach-voice-app

2. Create and activate a virtual environment:
   python -m venv .venv
   # Windows PowerShell:
   .\.venv\Scripts\Activate.ps1
   # macOS / Linux:
   source .venv/bin/activate

3. Install dependencies:
   pip install -r requirements.txt

4. Create a .env file (copy from .env.example and fill in your key):
   OPENAI_API_KEY=<your_key>

5. Run locally:
   python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

6. Open: http://127.0.0.1:8000


DEPLOYING TO AZURE
------------------
Prerequisites:
  - Azure CLI installed and logged in (az login)
  - Access to the cg_irvine resource group

Step 1 — Create the deployment zip (MUST use Python, not PowerShell Compress-Archive).
PowerShell's Compress-Archive uses Windows backslash paths which break Linux imports.

  python -c "
  import zipfile, os
  with zipfile.ZipFile('deploy.zip', 'w', zipfile.ZIP_DEFLATED) as zf:
      for dirpath, dirnames, filenames in os.walk('app'):
          dirnames[:] = [d for d in dirnames if d != '__pycache__']
          for filename in filenames:
              if filename.endswith('.pyc'):
                  continue
              filepath = os.path.join(dirpath, filename)
              arcname = filepath.replace(os.sep, '/')
              zf.write(filepath, arcname)
      zf.write('requirements.txt', 'requirements.txt')
  print('deploy.zip created')
  "

Step 2 — Deploy to Azure:
  az webapp deploy -g cg_irvine -n coachme-vg --src-path deploy.zip --type zip

  Note: The CLI may report "Site failed to start" due to a 10-minute warm-up timeout,
  but the app usually starts successfully. Verify with the health check below.

Step 3 — Verify deployment:
  curl https://coachme-vg.azurewebsites.net/api/health
  Expected: {"status":"ok","openai_configured":true}

    --startup-file "gunicorn -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000 app.main:app"


VIEWING LOGS
------------
Stream live logs:
  az webapp log tail -g cg_irvine -n coachme-vg

Download logs:
  az webapp log download -g cg_irvine -n coachme-vg --log-file logs.zip

The most useful log file inside the zip is:
  LogFiles/<instance>_default_docker.log   (gunicorn/app stdout)


RESTARTING THE APP
------------------
  az webapp restart -g cg_irvine -n coachme-vg


KNOWN ISSUES
------------
1. Windows zip path issue:
   Never use PowerShell Compress-Archive to create the deployment zip.
   It stores paths with backslashes (app\main.py) which on Linux are treated
   as literal filenames, not directory separators. Use the Python zipfile
   script in Step 1 above.

2. Deployment timeout false positive:
   The az webapp deploy command polls for 10 minutes. If the app takes longer
   to warm up (decompression + gunicorn boot), the CLI reports failure even
   though the app started successfully. Always confirm with the health check.
